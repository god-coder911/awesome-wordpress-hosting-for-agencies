# Contributing

Thanks for considering an addition to this list. PRs are accepted when the proposed vendor or tool meets the entry criteria stated in the README's "Contributing" section for this specific list.

## Adding a new vendor

1. Fork the repo.
2. Add your vendor in the correct sub-category section of `README.md` using the existing format:

   ```
   - [Vendor Name](https://vendor-site.com?utm_source=github-awesome-list&utm_medium=curated) — One-line description. Most-distinctive feature: <the single feature that makes this vendor non-substitutable>. [Read our comparison →](https://ourcomparison)
   ```

3. Keep the vendor list alphabetical within each sub-category unless the category has an intentional ranking.
4. Verify the vendor meets all bullet points in the "Contributing" section of `README.md`.
5. Open the PR with a title like `add: <Vendor Name>` and a 1-line body explaining the distinctive feature.

## Rejection criteria

PRs will be declined (or the vendor removed if already merged) when any of the following apply:

- Vendor is a white-label or resold version of an existing entry.
- No public pricing page or no way to verify the "most-distinctive feature" claim.
- Vendor is a private-only / enterprise-only tool with no published pricing and no public customer list.
- Affiliate-link stuffing in the description (the single UTM-tagged link to the vendor homepage is the only monetization permitted in this list).
- Description reads as marketing copy rather than a neutral honest description of the tool.

## Fixing errors

Pricing changes, discontinued products, or factually wrong "distinctive feature" claims — please file an issue with the source URL proving the correction, or submit a PR with the fix + source link in the PR body. Time-bound claims (promos, "free until X date") will be removed; this list sticks to structural facts.
